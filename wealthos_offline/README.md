# WealthOS Offline

تطبيق **Flutter** لإدارة الحياة المالية الشخصية، يعمل **دون اتصال بالإنترنت بنسبة 100%** ويحفظ كل البيانات محليًا على الجهاز (SQLite) مع تشفير الحقول الحساسة. الواجهة عربية بالكامل مع دعم RTL.

> هذا هو الإصدار الأول (MVP). لا يوجد Login؛ الحماية عبر PIN/بصمة. لا تُرسَل أي بيانات إلى أي خادم.

## المزايا المنفّذة (MVP)

- **Onboarding Wizard**: بناء الملف المالي خطوة بخطوة.
- **Dashboard**: صافي الثروة، الأصول، الالتزامات، السيولة، الدخل/المصروف الشهري، التدفق النقدي، معدل الادخار، نسبة الديون، مؤشر الصحة المالية، مخطط توزيع الأصول، منحنى تطور الثروة، الالتزامات القادمة.
- **الأصول / الالتزامات**: CRUD كامل، ربط الأصل بالتزام، تسجيل بيع، تسجيل دفعات سداد.
- **الدخل / المصروفات**: دعم الفترات التاريخية (Salary/Expense History) عبر السنوات.
- **المساهمات العائلية**: تظهر في التقارير ولا تدخل ضمن صافي الثروة.
- **Timeline + Audit Log**: كل تعديل يُسجَّل في المسار الزمني وسجل التدقيق.
- **Calculation Engine**: محرك حسابات مالية مستقل.
- **Reports**: ملخصات مالية + تصدير PDF.
- **Goals / Reminders**: أهداف مالية وتنبيهات محلية تُطلق إشعارات نظام حقيقية (flutter_local_notifications) دون أي اتصال بالشبكة.
- **Multi-Currency**: عملات متعددة بأسعار تحويل يدوية، والعرض بالعملة الأساسية.
- **Chat Assistant**: أوامر بلغة طبيعية (Rule-Based، بدون إنترنت).
- **Security**: PIN + بصمة + تشفير AES-256 للحقول الحساسة.
- **Backup/Restore**: ملف نسخ احتياطي مشفّر بكلمة مرور (محلي، بدون سحابة).

## البنية المعمارية

```
lib/
├── main.dart                 # نقطة الدخول + تهيئة التشفير و SQLite
├── app.dart                  # MaterialApp + RTL + العربية + الثيم
├── core/                     # ثوابت، ثيم، أدوات، أمان، ترجمة
│   ├── constants/enums.dart
│   ├── theme/                # الألوان + الثيم
│   ├── utils/                # التنسيق + نطاقات التواريخ
│   ├── security/             # EncryptionService (AES-256)
│   └── localization/         # النصوص العربية
├── data/
│   ├── database/             # AppDatabase — مخطط الجداول العشرين
│   ├── models/               # نماذج البيانات (toMap/fromMap)
│   └── repositories/         # طبقة الوصول للبيانات + تدقيق + مسار زمني
├── services/                 # المحرك الحسابي، العملات، النسخ، PIN، الشات، التقارير
├── providers/                # Riverpod providers (حقن التبعيات + الحالة)
└── presentation/
    ├── screens/              # الشاشات
    └── widgets/              # عناصر مشتركة
```

**Layers**: Models → Database → Repositories → Services (Calculation Engine) → Providers → Screens/Widgets.

**State Management**: [Riverpod](https://riverpod.dev). أي تعديل يزيد `refreshProvider` فتُعاد القراءات المرتبطة تلقائيًا.

## التشغيل

يتطلب **Flutter 3.27+**.

```bash
cd wealthos_offline
flutter pub get
flutter run          # على جهاز/محاكي أندرويد
flutter test         # تشغيل اختبارات المنطق
```

## ملفات ثنائية يجب توليدها محليًا

بعض ملفات منصّة أندرويد الثنائية لا يمكن تضمينها نصيًا في المستودع:

- `android/gradle/wrapper/gradle-wrapper.jar` — يُولَّد بفتح المشروع في Android Studio أو عبر `gradle wrapper`.
- أيقونات التطبيق (`mipmap`) — يستخدم التطبيق الأيقونة الافتراضية حاليًا.
- دعم iOS — يُضاف لاحقًا عبر `flutter create --platforms=ios .`.

> إن أردت البدء بسرعة: نفّذ `flutter create --platforms=android .` داخل المجلد مرّة واحدة لتوليد الملفات الثنائية الناقصة (يحافظ على مجلد `lib/` كما هو)، ثم أعد ضبط `AndroidManifest.xml`/`MainActivity` إن لزم.

## ملاحظات

- **العربية في PDF**: لعرض العربية داخل ملفات PDF يلزم تضمين خط عربي (TTF) في `assets/fonts`. يُصدَّر التقرير حاليًا بعناوين لاتينية وأرقام لضمان العمل دون خطوط إضافية، بينما تُعرض التقارير كاملةً بالعربية داخل التطبيق.
- **بدون إنترنت**: لا يطلب التطبيق إذن `INTERNET` في `AndroidManifest.xml` — إثباتًا لكونه Offline بالكامل.
